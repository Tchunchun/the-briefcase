# Inbox

Append-only capture for ideas, bugs, requests, and follow-ups.

## Usage

- Add one line per item.
- **Title must be 3–7 words.** Put longer context after ` — ` (em-dash).
- Do not prioritize or expand here.
- Mark an item `[-> architect review]` when the brief is created and ready for architect assessment.

## Entries

- [idea] Keyboard shortcuts for power users — add common shortcuts for navigation and search
- [bug] Search fails on empty query — returns 500 instead of empty result set
- [question] Email verification for onboarding — should onboarding require email verification before first use?
