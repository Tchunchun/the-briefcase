# Definition of Done

Use this checklist to determine whether a task or feature is actually complete.

## Task Done

- [ ] The related acceptance criteria in `brief.md` are satisfied.
- [ ] The work functions end-to-end in the target environment.
- [ ] Relevant automated tests are added or updated under `tests/`.
- [ ] The matching task in `tasks.md` is checked off.
- [ ] The matching row in `backlog.md` has the correct status.
- [ ] Any non-trivial blocker, caveat, or test outcome is noted.
- [ ] The work has been reviewed and accepted.

## Feature Done

- [ ] All acceptance criteria in `brief.md` are satisfied.
- [ ] All tasks in `tasks.md` are checked off.
- [ ] All related backlog rows are accurate and up to date.
- [ ] Review findings are resolved or explicitly accepted.
- [ ] Release notes are created if the feature ships.
